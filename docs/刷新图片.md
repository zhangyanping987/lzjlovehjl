# 刷新图片指南

本站相册的图片**不是**存在 GitHub 仓库里，而是保存在 `public/photos.json` 中的**外链地址**。  
百度等图床链接会随时间失效，需要你在本地重新搜图，再推送到 GitHub，网站才会更新。

---

## 一、什么时候需要刷新？

出现以下情况时，建议重新跑一遍搜图脚本：

- 球体上很多图显示「加载失败」
- 大图模式里图片打不开
- 距离上次更新已经很久（外链容易过期）
- 想换一批新的神图 / 壁纸

> **不会自动更新**：网站不会自己搜图，必须本地执行命令后 push。

---

## 二、最快流程（推荐）

在项目根目录打开终端，依次执行：

```bash
# 1. 进入项目目录
cd /path/to/lzjlovehjl

# 2. 重新搜图（默认 150 张，百度图片）
npm run fetch-photos -- --count 150

# 3. 本地预览（可选，确认图片正常）
npm run dev
# 浏览器打开 http://localhost:5173/lzjlovehjl/

# 4. 提交并推送到 GitHub
git add public/photos.json
git commit -m "refresh photo urls"
git push
```

推送成功后，GitHub Actions 会自动重新部署，约 **1～2 分钟** 后线上站点生效。

访问地址：

```
https://<你的GitHub用户名>.github.io/lzjlovehjl/
```

## 三、命令参数说明

查看完整帮助：

```bash
npm run fetch-photos -- --help
```

| 参数 | 说明 | 示例 |
|------|------|------|
| `--count <数量>` | 目标图片张数（默认 150） | `--count 200` |
| `--keyword <词>` | 只用一个搜索词 | `--keyword "贺峻霖 神图"` |
| `--keywords <a,b,c>` | 多个搜索词，逗号分隔 | `--keywords "贺峻霖 壁纸,贺峻霖 头像"` |
| `--from-file <文件>` | 从文件读取 URL，与搜图结果合并 | `--from-file urls.txt` |
| `--title <标题>` | 写入 JSON 的图片标题 | `--title "贺峻霖"` |
| `--demo` | 演示占位图（**勿用于正式站**） | `--demo` |

### 默认搜索关键词

脚本会自动使用以下关键词在百度图片搜图（可在 `--keywords` 中自定义覆盖）：

- 贺峻霖图片、贺峻霖 神仙颜值、贺峻霖 神图
- 贺峻霖 可爱、贺峻霖 壁纸高清、贺峻霖 高清 帅气
- 贺峻霖 头像、贺峻霖 可爱呆萌、贺峻霖 可爱霖霖兔
- 时代少年团 贺峻霖、TNT 贺峻霖

## 四、常用示例

### 默认刷新 150 张

```bash
npm run fetch-photos -- --count 150
```

### 多搜一些

```bash
npm run fetch-photos -- --count 200
```

### 指定关键词

```bash
npm run fetch-photos -- --keywords "贺峻霖 神图,贺峻霖 壁纸高清" --count 120
```

### 用自己收集的链接 + 自动搜图

1. 复制示例文件：

   ```bash
   cp urls.example.txt urls.txt
   ```

2. 编辑 `urls.txt`，每行一个图片 URL（`#` 开头为注释）

3. 运行：

   ```bash
   npm run fetch-photos -- --from-file urls.txt --count 150
   ```

> `urls.txt` 已在 `.gitignore` 中，不会被提交到 GitHub，适合放私人链接。

## 五、生成结果说明

- 输出文件：`public/photos.json`
- 格式示例：

  ```json
  [
    {
      "url": "https://img1.baidu.com/it/u=...",
      "title": "贺峻霖"
    }
  ]
  ```

- 网站启动时会读取该文件，浏览器**直接访问外链**加载图片
- 图片流量**不走 GitHub**，主要受外链图床是否可用影响

## 六、部署到 GitHub Pages

只需 push 含 `public/photos.json` 的提交，无需手动点发布。

1. 仓库需为 **Public（公开）**（免费版 GitHub Pages 要求）
2. **Settings → Pages → Source** 选 **GitHub Actions**
3. push 后在 **Actions** 查看 “Deploy to GitHub Pages” 是否成功

若只更新了图片、没改代码，只提交 `public/photos.json` 即可：

```bash
git add public/photos.json
git commit -m "refresh photo urls"
git push
```

## 七、常见问题

### 1. 搜图脚本报错 / 找到 0 张

- 检查网络能否访问百度图片
- 换关键词重试：`--keywords "贺峻霖 神图"`
- 使用 `--from-file urls.txt` 手动补充链接

### 2. 本地正常，线上部分图打不开

- 外链已失效 → 重新 `fetch-photos` 并 push
- 访客网络无法访问百度图床 → 尽量使用稳定图床链接，写入 `urls.txt`

### 3. push 后网站没变化

- 等 Actions 跑完（1～2 分钟）
- 浏览器 **强制刷新**（Ctrl+F5）或清缓存
- 确认 push 的是 `main` 分支且 Actions 无报错

### 4. 需要改代码后一起发布

```bash
git add .
git commit -m "update site"
git push
```

代码与 `photos.json` 会一起部署。
